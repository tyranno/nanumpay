### 1.1.1 지급액 산출 공식

**산출 방식**: 누적 배분 방식 (paymentCalculator.js)

```
계산 순서 (F1 → F2 → F3 → ... → F8):
1. 풀 금액 = 총 매출 × 해당 등급 배분율
2. 풀 대상자 = 현재 등급 인원 + 다음 등급 인원
3. 1인당 추가 금액 = 풀 금액 ÷ 풀 대상자
4. 누적 금액 = 이전 등급 추가 금액 + 현재 추가 금액
5. 회차당 금액 = 누적 금액 ÷ 10 (100원 단위 절삭)
```

#### 산출 예시 (매출 1억원, 각 등급 1명씩)

| 등급 | 배분율 | 풀 금액 | 풀 대상자 | 1인당 추가 | 누적 금액 | 회차당 금액 |
|------|--------|---------|----------|-----------|----------|------------|
| F1 | 24% | 24,000,000원 | 2명 (F1+F2) | 12,000,000원 | 12,000,000원 | **1,200,000원** |
| F2 | 19% | 19,000,000원 | 2명 (F2+F3) | 9,500,000원 | 21,500,000원 | **2,150,000원** |
| F3 | 14% | 14,000,000원 | 2명 (F3+F4) | 7,000,000원 | 16,500,000원 | **1,650,000원** |
| F4 | 9% | 9,000,000원 | 2명 (F4+F5) | 4,500,000원 | 11,500,000원 | **1,150,000원** |
| F5 | 5% | 5,000,000원 | 2명 (F5+F6) | 2,500,000원 | 7,000,000원 | **700,000원** |
| F6 | 3% | 3,000,000원 | 2명 (F6+F7) | 1,500,000원 | 4,000,000원 | **400,000원** |

### 1.3 등급 조건 계층 구조

등급은 **하위 등급의 존재**를 기반으로 결정됩니다. 상위 등급이 되려면 해당 하위 등급 요건을 충족해야 합니다.

```
F8 ← F7 3명 필요 (2:1 또는 1:2)
 │
F7 ← F6 3명 필요 (2:1 또는 1:2)
 │
F6 ← F5 3명 필요 (2:1 또는 1:2)
 │
F5 ← F4 3명 필요 (2:1 또는 1:2)
 │
F4 ← F3 각 1명 필요 (좌우 각각)
 │
F3 ← F2 각 1명 필요 (좌우 각각)
 │
F2 ← 좌우 자식 모두 보유
 │
F1 ← 자식 없음 또는 한쪽만
```

#### 실제 검증 예시: 김진숙1이 F6인 이유

**김진숙1 서브트리 구조**

| 위치 | F5 인원 | F4 인원 | F3 인원 | F2 인원 | F1 인원 |
|------|--------|--------|--------|--------|--------|
| 좌측 서브트리 | 3명 | 7명 | 14명 | 27명 | 59명 |
| 우측 서브트리 | 3명 | 9명 | 15명 | 32명 | 78명 |

**F6 조건 검증** (F5 총 3명, 2:1 또는 1:2 이상)
```
좌측 F5: 3명 (김진숙2, 박세환1, 공전식1)
우측 F5: 3명 (김태식1, 윤옥순1, 김보미1)
─────────────────────────────────
총 F5: 6명 (3:3 → 2:1 조건 충족 ✅)
결과: F6 조건 충족 ✅
```

**하위 등급 조건 연쇄 검증**

| 대상자 | 현재 등급 | 필요 조건 | 좌측 | 우측 | 판정 |
|--------|----------|----------|------|------|------|
| 김진숙1 | F6 | F5 3명 (2:1) | F5 3명 | F5 3명 | ✅ 충족 |
| 김진숙2 | F5 | F4 3명 (2:1) | F4 5명 | F4 2명 | ✅ 충족 (5:2) |
| 박세환2 | F4 | F3 각 1명 | F3 2명 | F3 1명 | ✅ 충족 |
| 선정화1 | F4 | F3 각 1명 | F3 1명 | F3 3명 | ✅ 충족 |

> **핵심**: 상위 등급은 하위 등급 인원의 충족을 전제로 합니다.
> 예: F6이 되려면 → F5 3명 필요 → 각 F5는 F4 3명 필요 → 각 F4는 F3 2명 필요 → ...

### 1.4 등급 계산 로직 검증

#### 계산 방식: 리프 노드 상향식 (Bottom-Up)
```
1. 모든 사용자를 조회하여 트리 맵 구성
2. 리프 노드(자식 없음)부터 시작
3. 부모 방향으로 순차적으로 등급 계산
4. 각 노드에서 서브트리 정보 수집 후 등급 판정
```

#### 검증 케이스

| 검증 항목 | 테스트 조건 | 예상 결과 | 실제 결과 | 판정 |
|----------|------------|----------|----------|------|

| F2 조건 | 좌우 자식 모두 등록 | 부모 → F2 | F2 | ✅ |
| F3 조건 | 좌우 서브트리에 F2 각 1명 이상 | 부모 → F3 | F3 | ✅ |
| F4 조건 | 좌우 서브트리에 F3 각 1명 이상 | 부모 → F4 | F4 | ✅ |
| F5 조건 | 좌우 서브트리에 F4 총 3명 (2:1 또는 1:2) | 부모 → F5 | F5 | ✅ |

### 4.2 엑셀 등록 시 계정 자동 생성

엑셀 대량 등록 시 **3가지 계정**이 자동 생성됩니다.

```
[Excel 업로드]
    │
    ├─ 1. User (트리 노드) 생성
    │     - 이름, 전화번호, 등급, 부모/자식 관계
    │
    ├─ 2. UserAccount (용역자 로그인) 생성 또는 연결
    │     - loginId: 이름 (예: 나우누리1)
    │     - 초기 비밀번호: 전화번호 뒤 4자리
    │     - 개인정보: 주민번호, 은행, 계좌번호
    │
    └─ 3. PlannerAccount (설계사 로그인) 생성 또는 연결
          - loginId: 설계사 이름
          - 초기 비밀번호: 전화번호 뒤 4자리 (없으면 0000)
          - 신규 설계사면 자동 생성
```

#### 계정 생성 규칙

| 항목 | 규칙 | 예시 |
|------|------|------|

## 5. 지급 금액 산출 기능

### 4.1 금액 산출 방식

시스템은 두 가지 금액 산출 방식을 지원합니다:

| 방식 | 설명 | 우선순위 |
|------|------|---------|
| **자동 계산** | 매출액과 등급별 인원을 기반으로 시스템이 자동 계산 | 기본값 |
| **수동 조정** | 관리자가 등급별 회차당 금액을 직접 지정 | 우선 적용 |

### 4.2 자동 계산 공식

#### 등급별 배분율 (코드 기준: paymentCalculator.js)
| 등급 | 배분율 | 풀 금액 (1억 기준) |
|------|--------|------------------|
| F1 | 24% | 24,000,000원 |
| F2 | 19% | 19,000,000원 |
| F3 | 14% | 14,000,000원 |
| F4 | 9% | 9,000,000원 |
| F5 | 5% | 5,000,000원 |
| F6 | 3% | 3,000,000원 |
| F7 | 2% | 2,000,000원 |
| F8 | 1% | 1,000,000원 |
| **합계** | **77%** | **77,000,000원** |

> **회사 운영비**: 23% (매출의 23%는 회사 운영비로 사용)

#### 누적 배분 계산 공식
```
각 등급별 계산 순서 (F1 → F2 → F3 → ... → F8):

1. 풀 금액 = 총 매출 × 해당 등급 배분율
2. 풀 대상자 = 현재 등급 인원 + 다음 등급 인원
3. 1인당 추가 금액 = 풀 금액 ÷ 풀 대상자
4. 최종 금액 = 이전 등급에서 넘어온 금액 + 1인당 추가 금액 (누적)
5. 회차당 금액 = 최종 금액 ÷ 10 (100원 단위 절삭)
```

#### 자동 계산 예시 (2025-11월 실제 데이터)

**조건**: 매출 80,000,000원, 등급 분포 F1=74명, F2=34명, F3=18명, F4=7명, F5=3명

| 등급 | 배분율 | 풀 금액 | 풀 대상자 | 1인당 추가 | 누적 금액 | 회차당 (자동) |
|------|--------|---------|----------|-----------|----------|--------------|
| F1 | 24% | 19,200,000원 | 108명 (F1:74+F2:34) | 177,778원 | 177,778원 | **17,800원** |
| F2 | 19% | 15,200,000원 | 52명 (F2:34+F3:18) | 292,308원 | 470,085원 | **47,000원** |
| F3 | 14% | 11,200,000원 | 25명 (F3:18+F4:7) | 448,000원 | 740,308원 | **74,000원** |
| F4 | 9% | 7,200,000원 | 10명 (F4:7+F5:3) | 720,000원 | 1,168,000원 | **116,800원** |
| F5 | 5% | 4,000,000원 | 3명 (F5:3) | 1,333,333원 | 2,053,333원 | **205,300원** |

> **참고**: 실제 지급액은 관리자가 수동 조정한 금액이 우선 적용됩니다. (아래 4.3 참조)

|------------|------|
| 등록 | 신규 용역자 등록 (계획 유형) |
| 승급 | 등급 상승 (계획 유형) |
| 기본지급 | 등록/승급 시 생성되는 최초 10회 지급 (지급 구분) |
| 추가지급 | 기본지급 완료 후 승급 없을 시 추가 10회 지급 (지급 구분) |
| 활성 | 현재 지급 진행 중 (상태) |
| 중단 | 승급으로 인해 추가지급 중단됨 (상태) |
| skip | 보험 미충족으로 해당 회차 지급 건너뜀 |
| 유예기간 | 승급 후 2개월간 보험 조건 면제. 승급→1개월 후 지급 시작→첫 지급은 무조건 진행→2개월까지 유예→3개월차부터 보험 필수 |
| 승계 | 1단계 승급 시 이전 보험 기준 인정 |
| 절삭 | 100원 미만 버림 (예: 23,456원 → 23,400원) |
